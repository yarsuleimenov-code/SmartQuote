Service Areas

Purpose: Service Areas define regional cost values used by route and access calculations.

Use zone-level values to manage toll fees, parking fees, ticket risk, and surcharge separately for each region.

If a regional component does not apply, keep its value at 0.

Purpose

References are admin-managed lookup tables used by the broker workflow and pricing engine.

References vs Variables

References define selectable business entities, such as vehicles, packaging types, item templates, service areas, and insurance types. Variables define numeric pricing inputs and calculation parameters.

Active / Inactive Logic

The Active flag controls whether a reference item is available for new quotes. Inactive records should remain stored for audit history and existing estimates.

Regional Rules

Service Areas contain regional charges such as toll fees, parking fees, ticket risk, and zone surcharge. If a component does not apply, use 0.

Best Practice

Do not delete reference records that may already be used in historical estimates. Use Active = false instead.
